GEJAST v422 config recovery

Root cause fixed:
- the shared gejast-config.js MutationObserver approach can self-trigger endlessly because applyVersionLabel mutates DOM/styles, which re-triggers the observer on the whole document
- that can cause Firefox to warn that the page is slowing down or hang on a blank screen even when network requests already finished

This recovery file:
- removes the observer-style behavior entirely
- keeps version ownership centralized in gejast-config.js
- applies the visible version label once on boot only
- does not auto-run the strict logged-out redirect globally
