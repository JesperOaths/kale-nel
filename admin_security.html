Security & hosting

Deze pagina vat samen wat in de bundle is dichtgezet en welke stap buiten de repo nog nodig is voor echte hosting-level bescherming.

Wat al in de bundle zit

  • Force-HTTPS redirect script op de pagina’s.
  • noindex,nofollow op admin-/vault-/leaderboard-pagina’s.
  • Admin-session gate op de gevoelige HTML-pagina’s.
  • Supabase admin-RPC’s blijven de echte backend-check doen.

Wat nog buiten de repo moet gebeuren

  • GitHub Pages: zet Enforce HTTPS aan in de Pages-instellingen voor het domein.
  • Voor échte afscherming: verhuis admin-tools naar een protected subdomain/path achter Cloudflare Access, Netlify Identity, Vercel middleware of een reverse proxy.
  • Laat de v62 SQL draaien zodat expired queue, allowed usernames en admin audit-routes actief worden.

Belangrijk

Deze statische repo kan de admin-pagina’s niet echt server-side verbergen zolang ze rechtstreeks via GitHub Pages worden geserveerd. De frontend-gates zijn nuttig, maar geen vervanging voor hosting-auth.
